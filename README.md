# Metro Kyiv

Встановлюваний PWA-навігатор метро й наземного транспорту Києва та області.
Застосунок збирається у звичайний каталог `dist`; базова навігація й збережена
транспортна мережа працюють офлайн, а Pages Functions проксіюють лише зовнішні
оперативні джерела.

[Відкрити Metro Kyiv](https://metro-kyiv.pages.dev/)

## Можливості

- усі 52 чинні станції трьох ліній;
- найкоротший маршрут, пересадки, кількість станцій та орієнтовний час;
- інтерактивна SVG-схема з підсвічуванням маршруту;
- масштабована SVG-схема з рознесеними підписами та чіткими пересадками;
- секундний розрахунковий таймер у двох напрямках для кожної станції;
- постійно відстежувана станція, пошук і таймери всіх 52 станцій;
- обрані станції та тема в `localStorage`;
- найближча станція за геолокацією;
- посилання на маршрут через `?from=…&to=…`;
- PWA-маніфест, service worker та офлайн-навігація;
- уточнення координат з офіційного GeoJSON API Києва з локальним fallback;
- клавіатурна навігація та `prefers-reduced-motion`.
- адресний маршрут Києвом і областю з профілями «найшвидше», «менше
  пересадок», «менше пішки» та «мій транспорт»;
- 265 маршрутів і 3 257 зупинок в офлайн-мережі: автобуси, тролейбуси,
  трамваї, 93 міські маршрутки та 14 ключових приміських напрямків;
- точна офіційна геометрія міських маршруток, GPS-маркери обраних маршрутів,
  покрокова поїздка й сторінки маршрутів/зупинок;
- детальна міська карта Esri, 3D та супутниковий режим із назвами;
- явна мобільна навігація «Карта / Маршрут / Транспорт / Зміни» без
  прихованих подвійних натискань;
- планування на конкретний час за офіційними GTFS-відправленнями та активний
  супровід поїздки з наступною пересадкою;
- контекстні сповіщення про зміни для обраного транспорту;
- маршрутизація у Web Worker, щоб карта й адресні поля не зависали.

Таймер використовує оприлюднені містом типові інтервали: у будні
2:30–3:30 хв у пікові години та 5–6 хв у міжпік, у вихідні — 6–7 хв.
Це розрахунок, а не оперативні дані диспетчерської системи.

## Локальний запуск

Потрібен Node.js `>=22.13.0`.

```bash
npm install
npm run dev
```

Production-перевірка:

```bash
npm run lint
npm run typecheck
npm run test:model
npm run test:ui
npm run test:e2e
npm run check:production
npm test
```

`npm run lint` запускає ESLint для TypeScript, React Hooks, Cloudflare
Functions і Node-скриптів. `npm run typecheck` перевіряє весь `app/**`,
`src/**` та `functions/**`.

Компонентні тести запускаються через Vitest + React Testing Library у
jsdom. Playwright перевіряє production-збірку в мобільному й desktop Chromium:
перехід до транспорту одним натисканням, повернення на карту, фільтр маршруток,
сторінку маршруту, згортання панелі, супутниковий шар і приміські дані.

## GitHub і Cloudflare Pages

GitHub workflow `Production quality` запускає lint/typecheck, build,
модельні й компонентні тести, Playwright E2E, bundle-check, smoke та Lighthouse.

Cloudflare Pages підключений до GitHub напряму і не потребує API-токена в
репозиторії. Налаштування Pages:

- Production branch: `main`
- Build command: `npm run build`
- Build output directory: `dist`
- Root directory: `/`

Після push у `main` Cloudflare сам збирає та публікує нову версію на
`https://metro-kyiv.pages.dev`.

### Змінні середовища

`REALTIME_UPSTREAM_URL` — необов’язковий URL офіційного GTFS Realtime
upstream. Якщо змінну не задано, застосунок використовує поточний сумісний
hostname із коду. Її можна додати в Cloudflare Pages → Settings →
Environment variables без зберігання токенів у GitHub.

Усі запити до зовнішніх джерел мають 7-секундний timeout. Geocoder
ідентифікує застосунок через `User-Agent` і `Referer`, кешує результати на
edge та показує атрибуцію OpenStreetMap.

## Дані

Міський електро- й автотранспорт побудований з Kyivpastrans GTFS Static,
маршрутки — з офіційного `routeTaxi` GeoJSON порталу відкритих даних Києва,
обласні напрямки — з реєстру КОДА. У UI кожен результат позначено як «GPS
наживо», «за розкладом», «офіційний реєстр» або «час орієнтовний». Для
міського GTFS застосунок зберігає точні хвилини відправлення; для реєстрових
напрямків без повного розкладу використовує чесно позначений інтервальний
розрахунок.

Оновити офлайн-мережу можна командою `npm run sync:transit`. Генератор
завантажує GTFS, додає всі сторінки ArcGIS `routeTaxi`, геометрію напрямків і
реєстрові приміські маршрути. Якщо зовнішній feed тимчасово недоступний,
production продовжує використовувати останній валідний snapshot у репозиторії.
